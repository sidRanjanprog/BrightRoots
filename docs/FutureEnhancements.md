# BrightRoots - Future Enhancements

## Purpose

This document stores ideas that are intentionally excluded from the MVP.

The goal is to prevent scope creep while preserving future opportunities.

A feature should only move from this document into the product after the MVP is completed and evaluated.

# Feature Evaluation Rule

Before implementing any future feature, ask:

1. Does it solve a real user problem?
2. Does it align with the BrightRoots vision?
3. Is the MVP already complete?
4. Can the feature's value be clearly explained?

If the answer is "No" to any of the above, postpone implementation.
