# BrightRoots - Database Design Document

## Overview

BrightRoots uses MongoDB as its primary database.

The database is designed to support:

- User Authentication
- Child Profile Management
- Habit Tracking
- Personalized Recommendations
- Educational Resources

The design prioritizes:

- Simplicity
- Scalability
- Maintainability
- Future feature expansion

---

# Collections Overview

The BrightRoots MVP consists of the following collections:

1. Users
2. Children
3. ScreenTimeLogs
4. SleepLogs
5. OutdoorActivityLogs
6. Recommendations
7. Resources

---

# 1. Users Collection

## Purpose

Stores parent account information and authentication details.

## Fields


| Field     | Type            | Required |
| --------- | --------------- | -------- |
| _id       | ObjectId        | Yes      |
| fullName  | String          | Yes      |
| email     | String          | Yes      |
| password  | String (hashed) | Yes      |
| createdAt | Date            | Yes      |
| updatedAt | Date            | Yes      |


## Relationships

One User can have multiple Children.

Relationship:

```text
User
  │
  └── Children (1:N)

```

---

# 2. Children Collection

## Purpose

Stores child profile information.

## Fields


| Field     | Type     | Required |
| --------- | -------- | -------- |
| _id       | ObjectId | Yes      |
| parentId  | ObjectId | Yes      |
| name      | String   | Yes      |
| age       | Number   | Yes      |
| gender    | String   | Yes      |
| createdAt | Date     | Yes      |
| updatedAt | Date     | Yes      |


## Relationships

Each Child belongs to one User.

Each Child can have:

- Screen Time Logs
- Sleep Logs
- Outdoor Activity Logs
- Recommendations

# Indexing Strategy

## Users

Create unique index:

```text
email

```

Purpose:

- Fast login
- Prevent duplicate accounts

---

## Children

Create index:

```text
parentId

```

Purpose:

- Fast retrieval of children for a parent

---

## ScreenTimeLogs

Create index:

```text
childId
date

```

Purpose:

- Fast habit tracking queries

---

## SleepLogs

Create index:

```text
childId
date

```

Purpose:

- Fast sleep history retrieval

---

## OutdoorActivityLogs

Create index:

```text
childId
date

```

Purpose:

- Fast activity history retrieval
